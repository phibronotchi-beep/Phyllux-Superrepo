# Partnership Workflow

**Purpose:** Complete workflow for partnership development from opportunity identification to closed deal.

**Last Updated:** January 25, 2026

---

## Workflow Stages

![Partnership Lifecycle](visuals/workflows/partnership-lifecycle.svg)

### Stage 1: Opportunity Identification
- Review TOP_5_LANES.md
- Check PARTNER_TARGET_LIST.md
- Score in OPPORTUNITY_MATRIX.md
- Determine tier assignment

### Stage 2: Brief Generation
- Select tier template
- Customize for partner
- Add technical validation
- Review for qualified language

### Stage 3: Initial Contact
- Send brief via email
- Include public repo links
- Set follow-up expectation
- Log in pipeline

### Stage 4: Follow-Up
- Follow up after appropriate timeframe
- Schedule discussion call
- Provide additional information
- Update pipeline status

### Stage 5: Technical Discussion
- Review technical validation
- Discuss integration opportunities
- Explore collaboration
- Plan next steps

### Stage 6: Negotiation
- Review licensing framework
- Negotiate terms
- Document agreement
- Update pipeline

### Stage 7: Agreement Execution
- Finalize agreement
- Execute contract
- Log in closed_deals.md
- Begin execution
