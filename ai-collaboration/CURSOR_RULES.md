# Cursor Rules

**Purpose:** AI safety guardrails for Cursor AI collaboration. Critical rules to prevent IP damage and maintain qualified language.

**Last Updated:** January 25, 2026

---

## CRITICAL: DO NOT MODIFY

**Under NO circumstances should AI modify these:**
- LICENSE files (in any repo)
- PRIOR_ART.md, PATENTS.md, DISCLOSURE.md files
- INVENTORSHIP_DECLARATION.md
- TIMESTAMP.md
- Maturity tags in phyllux-inventions-wip
- Commit history or git tags
- Any file in legal/ directories

---

## Qualified Language Requirements

**ALWAYS use qualified language:**
- "may provide"
- "designed to"
- "simulations suggest"
- "could enable"
- "appears to"

**NEVER use absolute claims:**
- "guarantees"
- "proven"
- "definitely"
- Absolute performance assertions

## Safe Modification Zones

**Safe to modify in superrepo:**
- CONTROL_PANEL.md (daily updates)

- Partner briefs (using templates)

- Maturity tags
