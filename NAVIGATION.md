## üîç By File Type

### Markdown Documentation
- All `.md` files contain structured documentation
- Use qualified language ("may", "designed to", "simulations suggest")
- Cross-reference other files using relative paths

### Python Scripts
- All `.py` files in `analytics/` are functional stubs
- Include detailed docstrings
- Mark TODOs clearly
- Read-only by design (don't modify source repos)

### Configuration Files
- `.gitignore` - Excludes confidential pipeline data
- `.github/workflows/` - CI/CD automation
- `.cursor/rules/` - AI safety guardrails

### PPA Filing Workflow
1. Review [PPA_EVIDENCE_MAP.md](strategy/PPA_EVIDENCE_MAP.md)
2. Run [ppa_claim_mapper.py](analytics/ppa_claim_mapper.py)
3. Run [embodiment_generator.py](analytics/embodiment_generator.py)
4. Review [legal/PPA_SUPPORT_DOCS.md](legal/PPA_SUPPORT_DOCS.md)
5. Coordinate with attorney

### Partnership Workflow
1. Identify opportunity in [TOP_5_LANES.md](strategy/TOP_5_LANES.md)
2. Check [PARTNER_TARGET_LIST.md](strategy/PARTNER_TARGET_LIST.md)
3. Select tier template from [partnerships/briefs/](partnerships/briefs/)
4. Follow [FAST_TRACK_WORKFLOW.md](partnerships/FAST_TRACK_WORKFLOW.md)
5. Generate brief using [brief_generator.py](analytics/brief_generator.py)
6. Log in [pipeline/active_conversations.md](partnerships/pipeline/active_conversations.md)

### Daily Operations Workflow
1. Open [CONTROL_PANEL.md](CONTROL_PANEL.md)
2. Pick today's priority
3. Complete 15-minute task
4. Update status
5. Plan tomorrow

### Treatment Handoff Workflow
1. Review [HANDOFF_GUIDE.md](HANDOFF_GUIDE.md) with helper
2. Mark pre-approved briefs as "‚úÖ Ready"
3. Set up email forwarding
